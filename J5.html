💬 Questions fréquentes en soutenance

Prépare des réponses courtes et précises à ces questions — elles reviennent souvent.

Les documents Firestore sont limités à 1 Mo. Un tableau dans le document grossit indéfiniment avec chaque partie jouée. La sous-collection évite cette limite, permet des requêtes ciblées (tri par date, limite à 10), et s'aligne avec les règles de sécurité (chaque utilisateur n'accède qu'à son sous-arbre).

onSnapshot ouvre une connexion WebSocket persistante. Si on ne la ferme pas quand le composant est démonté, la callback continue de s'exécuter en mémoire — fuite mémoire garantie. return unsub dans useEffect est le mécanisme de cleanup de React : il appelle cette fonction automatiquement au démontage.

Via initializeAuth avec getReactNativePersistence(AsyncStorage). Firebase stocke le token d'authentification dans AsyncStorage au lieu de la mémoire. Au redémarrage, onAuthStateChanged le retrouve automatiquement et restaure la session sans nouvelle connexion.

Les règles de sécurité Firestore, évaluées côté serveur. La clé API (apiKey dans firebaseConfig) n'est pas secrète — elle identifie le projet Firebase mais ne donne pas accès sans règles appropriées. Les règles vérifient request.auth.uid pour s'assurer que chaque utilisateur ne peut accéder qu'à ses propres données.

getDocs fait une lecture ponctuelle — utilisé pour charger les questions du quiz (données statiques qui ne changent pas pendant la partie). onSnapshot maintient une connexion persistante — utilisé pour l'historique dans ProfileScreen, pour que les nouveaux scores apparaissent en temps réel sans recharger.

QCM Bilan — 15 questions sur les 5 jours

Révision complète J1→J5 · Composants · Navigation · Firebase · UX · Build

📝Révision finale couvrant l'ensemble du cours. Sélectionne une réponse par question puis clique sur "Valider mes réponses".
Q1 / 15 · J1 — Composants
En React Native, pourquoi ne peut-on pas écrire du texte directement dans un <View> ?
Contrairement au web où un div peut contenir du texte, React Native compile vers des vues natives. Les moteurs iOS/Android ne savent pas afficher du texte dans un conteneur générique. Tout texte doit être dans <Text> — c'est une règle absolue, pas une convention.
Q2 / 15 · J1 — useState
Que se passe-t-il si on écrit directement count = count + 1 au lieu de setCount(count + 1) ?
React surveille les changements d'état uniquement via le setter (setCount). Muter directement la variable ne notifie pas React — la valeur change en mémoire mais l'UI reste gelée sur l'ancienne valeur. Le setter est le seul moyen de déclencher un re-render.
Q3 / 15 · J1 — Flexbox
Quelle est la direction par défaut de Flexbox en React Native ?
C'est une différence fondamentale avec CSS web. En React Native, flexDirection est "column" par défaut — les éléments s'empilent de haut en bas. En CSS web, c'est "row" (horizontal). Pour mettre des éléments côte à côte en RN, il faut explicitement ajouter flexDirection: 'row'.
Q4 / 15 · J2 — Navigation
Pourquoi utiliser navigation.replace() après une connexion réussie plutôt que navigate() ?
Après une connexion réussie, on ne veut pas que l'utilisateur puisse appuyer "Retour" et retomber sur l'écran de login. navigate() empilerait l'app sur le login — le bouton retour Android/iOS ramènerait sur le login. replace() remplace dans la pile — le login n'est plus accessible.
Q5 / 15 · J2 — useEffect
Qu'est-ce qui se passe si on oublie le tableau de dépendances [] dans useEffect ?
Sans tableau de dépendances, useEffect tourne après chaque render. Si le code dans useEffect appelle setData() (ce qui cause un re-render), ça déclenche un nouveau useEffect, qui appelle setData()… boucle infinie. Les tableaux de dépendances ne sont pas optionnels — ils contrôlent quand l'effet se déclenche.
Q6 / 15 · J2 — Architecture
Dans App Quiz, pourquoi sépare-t-on AuthStack et AppStack en deux navigateurs distincts plutôt qu'un seul ?
La séparation AuthStack/AppStack est un pattern de sécurité. Un utilisateur non connecté ne doit jamais pouvoir naviguer vers les écrans de l'app (même via le bouton retour). En commutant entre deux Stacks distincts via onAuthStateChanged, on garantit que les écrans protégés ne sont jamais dans la pile de navigation d'un utilisateur non authentifié.
Q7 / 15 · J3 — Firebase Auth
Quelle est la première chose à faire après createUserWithEmailAndPassword() dans App Quiz ?
createUserWithEmailAndPassword() crée le compte ET connecte l'utilisateur automatiquement. onAuthStateChanged() détecte la connexion et commute vers AppStack sans action supplémentaire. La seule chose à faire est de créer le profil Firestore (setDoc dans users/uid) pendant qu'on a accès à user.uid.
Q8 / 15 · J3 — Firestore
Quelle est la différence entre setDoc() et updateDoc() ?
setDoc() est destructif — il remplace tout le document. Si le document avait un champ "score" et que tu setDoc() avec juste un champ "pseudo", le champ "score" disparaît. updateDoc() est chirurgical — il ne touche qu'aux champs passés. Pour modifier le pseudo sans effacer le reste, toujours utiliser updateDoc().
Q9 / 15 · J3 — Sécurité
Pourquoi la collection questions est en lecture seule (allow write: if false) dans les règles Firestore ?
Les questions sont gérées par l'admin (via console Firebase ou script de seed). Si l'écriture était ouverte, un utilisateur malveillant pourrait supprimer toutes les questions, modifier les bonnes réponses, ou injecter du contenu. allow write: if false garantit que les données ne peuvent être modifiées que par l'admin via la console.
Q10 / 15 · J4 — AsyncStorage
Tu stockes un objet {'{ nom: "Alice", score: 42 }'} dans AsyncStorage. Comment le récupérer correctement ?
AsyncStorage stocke uniquement des strings. Quand tu écris setItem('@user', JSON.stringify(obj)), la valeur stockée est la string '{"nom":"Alice","score":42}'. Pour récupérer l'objet, il faut parser cette string avec JSON.parse(). Ne pas oublier de gérer le cas null (clé absente) : JSON.parse(valeur || 'null').
Q11 / 15 · J4 — UX
Pourquoi désactiver le bouton "Soumettre" pendant une requête Firebase en cours ?
Le double-submit est un bug classique — l'utilisateur tape deux fois rapidement sur "S'inscrire" et deux comptes sont créés, ou le score est sauvegardé deux fois. Pattern standard : setLoading(true) au début de la requête + disabled={loading} sur le bouton + setLoading(false) dans finally.
Q12 / 15 · J4 — EAS Build
Quelle commande EAS génère un APK directement installable sur Android (pas pour le Play Store) ?
Le profil "preview" dans eas.json est configuré avec buildType: "apk" — il génère un APK directement installable. Le profil "production" génère un AAB (Android App Bundle) requis par le Play Store mais non installable directement. expo build:android est l'ancienne commande dépréciée (Expo CLI classique).
Q13 / 15 · J5 — Mini-app
Tu dois réaliser la mini-app en 3h. Quelle est la meilleure stratégie pour maximiser ta note ?
La stratégie optimale : Auth (base de tout, bloquant si absent) → CRUD Firestore (cœur du sujet) → UX (finitions qui améliorent la note) → Bonus (seulement si le reste est complet). Push régulier = sauvegarde contre les pannes. Un projet qui tourne avec 3 fonctionnalités bien faites > un projet ambitieux qui crashe.
Q14 / 15 · Transversal
Dans App Quiz, quels composants doivent obligatoirement avoir un cleanup dans leur useEffect ?
Les listeners Firebase (onSnapshot, onAuthStateChanged) ouvrent des connexions WebSocket persistantes. Sans cleanup (return unsub), ces connexions restent actives après le démontage du composant — les callbacks continuent de s'exécuter, causant des setState sur des composants démontés et des fuites mémoire. Les getDocs() ponctuels n'ont pas besoin de cleanup.
Q15 / 15 · Transversal
Quelle est la différence fondamentale entre React Native et React (web) ?
La différence fondamentale : React web cible le DOM HTML (navigateur). React Native cible les composants natifs iOS et Android. Les hooks, la syntaxe JSX, la logique d'état et le fetch sont identiques — seuls les composants de base changent. View = div, Text = p/span, Image = img, TouchableOpacity = button, FlatList = ul/li.

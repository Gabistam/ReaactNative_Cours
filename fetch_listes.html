🎯 Ce qu'on va construire
💡 Vue d'ensemble

Une app à 3 onglets en bas de l'écran. Chaque onglet est un écran indépendant qui va chercher des données différentes sur internet et les affiche avec un type de liste différent. C'est exactement la structure d'une vraie app : plusieurs sections, chacune avec son propre contenu et sa propre façon de l'afficher.

// ─── IMPORTS ────────────────────────────────────────────────────────────────

// Une seule fois dans toute l'app, ici dans App.js
import { NavigationContainer } from '@react-navigation/native';

// Notre Tab Navigator défini dans navigation/TabNavigator.js
import { TabNavigator } from './navigation/TabNavigator';

export default function App() {
  return (
    // NavigationContainer entoure TOUT — c'est lui qui permet
    // à n'importe quel écran enfant d'utiliser navigation.navigate()
    <NavigationContainer>
      <TabNavigator />
    </NavigationContainer>
  );
}

Navigation entre les 3 pages

Tab Navigator avec icônes — un onglet par type de liste

🗂 TabNavigator.js — les 3 onglets
1

Pourquoi un Tab Navigator plutôt qu'un Stack ?

Les 3 pages ne sont pas liées par un parcours ("d'abord ça, puis ça") — elles sont indépendantes. C'est exactement le cas d'usage du Tab Navigator : plusieurs sections accessibles à tout moment, sans ordre imposé.

2

Chaque onglet garde son état en mémoire

Une fois qu'un onglet a chargé ses données (fetch terminé), passer à un autre onglet puis revenir ne relance pas le fetch — l'écran est resté monté en arrière-plan.

navigation/TabNavigator.js
// ─── IMPORTS ────────────────────────────────────────────────────────────────
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
// Ionicons : bibliothèque d'icônes incluse avec Expo
import { Ionicons } from '@expo/vector-icons';

// On importe les 3 écrans qu'on va créer dans les onglets suivants
import ArticlesScreen from '../screens/ArticlesScreen';
import TachesScreen   from '../screens/TachesScreen';
import ProfilScreen   from '../screens/ProfilScreen';

// createBottomTabNavigator() retourne un objet avec Tab.Navigator et Tab.Screen
const Tab = createBottomTabNavigator();

export function TabNavigator() {
  return (
    <Tab.Navigator
      // screenOptions : fonction appelée pour CHAQUE onglet
      // "route" contient route.name = le nom de l'onglet courant
      screenOptions={({ route }) => ({

        // tabBarIcon : quelle icône afficher pour cet onglet
        // focused = true si l'onglet est actif
        tabBarIcon: ({ focused, color, size }) => {
          const icons = {
            Articles: focused ? 'newspaper'      : 'newspaper-outline',
            Taches:   focused ? 'checkbox'       : 'checkbox-outline',
            Profil:   focused ? 'person'         : 'person-outline',
          };
          return <Ionicons name={icons[route.name]} size={size} color={color} />;
        },

        // Couleurs de la tab bar
        tabBarActiveTintColor:   '#3b82f6',
        tabBarInactiveTintColor: '#64748b',
        tabBarStyle: { backgroundColor: '#1e293b', borderTopColor: '#334155' },

        // Header en haut de chaque écran
        headerStyle:     { backgroundColor: '#0f172a' },
        headerTintColor: '#f1f5f9',
      })}
    >
      // Chaque Tab.Screen = un onglet · name = utilisé pour l'icône au-dessus
      // options.title = le texte affiché sous l'icône et dans le header
      <Tab.Screen name="Articles" component={ArticlesScreen} options={{ title: 'Articles' }} />
      <Tab.Screen name="Taches"   component={TachesScreen}   options={{ title: 'Tâches' }}   />
      <Tab.Screen name="Profil"   component={ProfilScreen}   options={{ title: 'Profil' }}   />
    </Tab.Navigator>
  );
}

screens/ArticlesScreen.js
import { useState, useEffect } from 'react';
import {
  View, Text, FlatList,
  ActivityIndicator, StyleSheet,
} from 'react-native';

export default function ArticlesScreen() {
  // ─── LES 3 ÉTATS DU FETCH ──────────────────────────────────────────────
  // articles : tableau des articles · loading : chargement en cours ?
  // error : message d'erreur ou null si tout va bien
  const [articles, setArticles] = useState([]);
  const [loading,  setLoading]  = useState(true);
  const [error,    setError]    = useState(null);

  // useEffect avec [] : ne s'exécute qu'une fois, au montage de l'écran
  useEffect(() => {
    const fetchArticles = async () => {
      try {
        const response = await fetch('https://jsonplaceholder.typicode.com/posts');
        if (!response.ok) throw new Error('Erreur ' + response.status);
        const data = await response.json();
        setArticles(data);
      } catch (err) {
        setError('Impossible de charger les articles.');
      } finally {
        setLoading(false);  // dans tous les cas, on arrête le spinner
      }
    };
    fetchArticles();
  }, []);

  // CAS 1 : chargement → spinner centré
  if (loading) return (
    <View style={[styles.container, styles.center]}>
      <ActivityIndicator size="large" color="#3b82f6" />
    </View>
  );

  // CAS 2 : erreur → message rouge
  if (error) return (
    <View style={[styles.container, styles.center]}>
      <Text style={styles.error}>⚠ {'{'}error{'}'}</Text>
    </View>
  );

  // CAS 3 : données prêtes → FlatList
  return (
    <View style={styles.container}>
      <FlatList
        // data : le tableau à afficher
        data={articles}
        // keyExtractor : clé unique par ligne (obligatoire)
        keyExtractor={(item) => item.id.toString()}
        // ListHeaderComponent : en-tête qui défile AVEC la liste
        ListHeaderComponent={
          <Text style={styles.header}>📰 Articles ({'{'}articles.length{'}'})</Text>
        }
        // renderItem : comment dessiner CHAQUE article
        renderItem={({ item }) => (
          <View style={styles.card}>
            <Text style={styles.cardTitle}>{'{'}item.id{'}'} · {'{'}item.title{'}'}</Text>
            {/* numberOfLines={2} : tronque à 2 lignes max */}
            <Text style={styles.cardBody} numberOfLines={2}>{'{'}item.body{'}'}</Text>
          </View>
        )}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#0f172a' },
  center:    { justifyContent: 'center', alignItems: 'center' },
  header:    { fontSize: 18, fontWeight: '700', color: '#f1f5f9', padding: 12 },
  error:     { color: '#fca5a5', fontSize: 15 },
  card:      { backgroundColor: '#1e293b', borderRadius: 10, padding: 14, marginHorizontal: 12, marginBottom: 10, borderLeftWidth: 3, borderLeftColor: '#3b82f6' },
  cardTitle: { fontSize: 14, fontWeight: '700', color: '#e2e8f0', marginBottom: 4 },
  cardBody:  { fontSize: 12.5, color: '#94a3b8', lineHeight: 18 },
});

💻 TachesScreen.js — code complet commenté
1

Étape 1 — récupérer les données à plat

L'API renvoie un simple tableau de tâches, sans groupement. C'est à nous de les organiser.

2

Étape 2 — transformer en sections

On sépare les tâches en deux groupes selon completed, puis on construit le tableau de sections attendu par SectionList.

screens/TachesScreen.js
import { useState, useEffect } from 'react';
import {
  View, Text, SectionList,
  ActivityIndicator, StyleSheet,
} from 'react-native';

export default function TachesScreen() {
  // On stocke directement les SECTIONS (déjà groupées), pas le tableau brut
  const [sections, setSections] = useState([]);
  const [loading,  setLoading]  = useState(true);
  const [error,    setError]    = useState(null);

  useEffect(() => {
    const fetchTaches = async () => {
      try {
        // On limite à 30 tâches avec _limit pour ne pas surcharger l'écran
        // (l'API en a 200 par défaut)
        const response = await fetch(
          'https://jsonplaceholder.typicode.com/todos?_limit=30'
        );
        if (!response.ok) throw new Error('Erreur ' + response.status);
        const data = await response.json();  // tableau plat de 30 tâches

        // ─── GROUPEMENT EN SECTIONS ──────────────────────────────────────
        // .filter() garde uniquement les tâches qui correspondent à la condition
        const aFaire    = data.filter((tache) => !tache.completed);
        const terminees = data.filter((tache) => tache.completed);

        // On construit le tableau de sections attendu par SectionList :
        // [{ title, data }, { title, data }]
        setSections([
          { title: `⏳ À faire (${'{'}aFaire.length{'}'})`,    data: aFaire },
          { title: `✅ Terminées (${'{'}terminees.length{'}'})`, data: terminees },
        ]);

      } catch (err) {
        setError('Impossible de charger les tâches.');
      } finally {
        setLoading(false);
      }
    };
    fetchTaches();
  }, []);

  if (loading) return (
    <View style={[styles.container, styles.center]}>
      <ActivityIndicator size="large" color="#7c3aed" />
    </View>
  );

  if (error) return (
    <View style={[styles.container, styles.center]}>
      <Text style={styles.error}>⚠ {'{'}error{'}'}</Text>
    </View>
  );

  return (
    <SectionList
      style={styles.container}
      // sections : le tableau [{ title, data }] qu'on a construit
      sections={sections}
      keyExtractor={(item) => item.id.toString()}

      // renderSectionHeader : dessine le TITRE de chaque section
      // { section } donne accès au "title" de la section courante
      renderSectionHeader={({ section }) => (
        <Text style={styles.sectionTitle}>{'{'}section.title{'}'}</Text>
      )}

      // renderItem : dessine CHAQUE tâche (comme dans FlatList)
      renderItem={({ item }) => (
        <View style={styles.todoRow}>
          {/* Pastille colorée selon l'état completed */}
          <View style={[
            styles.checkbox,
            item.completed && styles.checkboxDone,  // style conditionnel
          ]} />
          <Text style={styles.todoText} numberOfLines={1}>
            {'{'}item.title{'}'}
          </Text>
        </View>
      )}
    />
  );
}

const styles = StyleSheet.create({
  container:     { flex: 1, backgroundColor: '#0f172a' },
  center:        { justifyContent: 'center', alignItems: 'center' },
  error:         { color: '#fca5a5' },
  sectionTitle:  { fontSize: 13, fontWeight: '700', color: '#c4b5fd', backgroundColor: '#0f172a', padding: 10, paddingTop: 16 },
  todoRow:       { flexDirection: 'row', alignItems: 'center', gap: 10, backgroundColor: '#1e293b', marginHorizontal: 12, marginBottom: 6, padding: 10, borderRadius: 8 },
  checkbox:      { width: 16, height: 16, borderRadius: 4, borderWidth: 1.5, borderColor: '#475569' },
  checkboxDone:  { backgroundColor: '#34d399', borderColor: '#34d399' },
  todoText:      { color: '#cbd5e1', fontSize: 13, flex: 1 },
});
⚠️Piège fréquent : oublier que SectionList attend sections (pluriel) et pas data comme FlatList. La structure attendue est précise : [{'{'} title, data: [...] {'}'}].

Page 3 — Profil ScrollView

Fetch /users/1 · un seul objet, pas une liste

🎯 Pourquoi ScrollView ici (et pas FlatList) ?

Cette fois, l'API renvoie un seul objet (un utilisateur), pas un tableau d'éléments similaires. Il n'y a rien à "lister" — juste plusieurs blocs d'informations différents à empiler verticalement. C'est le cas d'usage typique de ScrollView.

💡Règle de décision : si tu réponds "j'affiche le MÊME composant plusieurs fois avec des données différentes" → FlatList/SectionList. Si tu réponds "j'affiche des informations différentes une seule fois chacune" → ScrollView.

💻 ProfilScreen.js — code complet commenté
1

Différence avec les pages précédentes

Pas de tableau, pas de keyExtractor, pas de renderItem. L'état user contient directement un objet, pas un tableau. On accède à ses champs avec user.name, user.email, etc.

2

Structure imbriquée de la réponse API

L'objet utilisateur contient lui-même des objets imbriqués : user.address.city, user.address.geo.lat. Il faut bien suivre la hiérarchie des accolades pour accéder à la bonne donnée.

screens/ProfilScreen.js
import { useState, useEffect } from 'react';
import {
  View, Text, ScrollView,
  ActivityIndicator, StyleSheet,
} from 'react-native';

export default function ProfilScreen() {
  // user : null au départ, puis l'objet utilisateur complet une fois chargé
  // (PAS un tableau — un seul objet)
  const [user,    setUser]    = useState(null);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    const fetchUser = async () => {
      try {
        // /users/1 : on demande l'utilisateur n°1 précisément
        // (au lieu de /users qui renverrait TOUS les utilisateurs)
        const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
        if (!response.ok) throw new Error('Erreur ' + response.status);
        const data = await response.json();  // UN objet, pas un tableau
        setUser(data);
      } catch (err) {
        setError('Impossible de charger le profil.');
      } finally {
        setLoading(false);
      }
    };
    fetchUser();
  }, []);

  if (loading) return (
    <View style={[styles.container, styles.center]}>
      <ActivityIndicator size="large" color="#3b82f6" />
    </View>
  );

  if (error) return (
    <View style={[styles.container, styles.center]}>
      <Text style={styles.error}>⚠ {'{'}error{'}'}</Text>
    </View>
  );

  // ─── CAS 3 : user contient l'objet complet ────────────────────────────
  // On accède directement aux champs : user.name, user.email, user.address...
  return (
    <ScrollView style={styles.container} contentContainerStyle={{ padding: 16 }}>

      {/* Carte d'identité */}
      <View style={styles.avatarCard}>
        <View style={styles.avatar}>
          {/* Initiales : 1ère lettre du prénom + 1ère lettre du nom de famille
              .split(' ') sépare "Leanne Graham" en ['Leanne', 'Graham']
              [0][0] prend la 1ère lettre du 1er mot */}
          <Text style={styles.avatarText}>
            {'{'}user.name.split(' ')[0][0]{'}'}{'{'}user.name.split(' ')[1]?.[0] ?? ''{'}'}
          </Text>
        </View>
        <Text style={styles.name}>{'{'}user.name{'}'}</Text>
        <Text style={styles.username}>@{'{'}user.username{'}'}</Text>
      </View>

      {/* Section Contact */}
      <Text style={styles.sectionTitle}>📧 Contact</Text>
      <View style={styles.infoBox}>
        <Text style={styles.infoText}>{'{'}user.email{'}'}</Text>
      </View>
      <View style={styles.infoBox}>
        <Text style={styles.infoText}>📞 {'{'}user.phone{'}'}</Text>
      </View>

      {/* Section Adresse — accès aux champs imbriqués user.address.xxx */}
      <Text style={styles.sectionTitle}>📍 Adresse</Text>
      <View style={styles.infoBox}>
        <Text style={styles.infoText}>
          {'{'}user.address.street{'}'}, {'{'}user.address.suite{'}'}{'\n'}
          {'{'}user.address.city{'}'}, {'{'}user.address.zipcode{'}'}
        </Text>
      </View>

      {/* Section Entreprise */}
      <Text style={styles.sectionTitle}>🏢 Entreprise</Text>
      <View style={styles.infoBox}>
        <Text style={styles.infoText}>{'{'}user.company.name{'}'}</Text>
      </View>

    </ScrollView>
  );
}

const styles = StyleSheet.create({
  container:    { flex: 1, backgroundColor: '#0f172a' },
  center:       { justifyContent: 'center', alignItems: 'center' },
  error:        { color: '#fca5a5' },
  avatarCard:   { alignItems: 'center', backgroundColor: '#1e293b', borderRadius: 12, padding: 20, marginBottom: 16 },
  avatar:       { width: 64, height: 64, borderRadius: 32, backgroundColor: '#3b82f6', justifyContent: 'center', alignItems: 'center', marginBottom: 10 },
  avatarText:   { color: '#fff', fontSize: 22, fontWeight: '700' },
  name:         { color: '#f1f5f9', fontSize: 17, fontWeight: '700' },
  username:     { color: '#64748b', fontSize: 13, marginTop: 2 },
  sectionTitle: { color: '#93c5fd', fontSize: 13, fontWeight: '700', marginBottom: 8, marginTop: 6 },
  infoBox:      { backgroundColor: '#1e293b', borderRadius: 8, padding: 12, marginBottom: 8 },
  infoText:     { color: '#cbd5e1', fontSize: 13, lineHeight: 19 },
});
⚠️Piège fréquent : oublier le ?. (optional chaining) sur user.name.split(' ')[1]?.[0]. Si le nom n'a qu'un seul mot, [1] serait undefined et [0] sur undefined ferait planter l'app. ?? '' donne une valeur de secours.

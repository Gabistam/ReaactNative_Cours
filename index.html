Comprendre React Native

Architecture · Différences Web/Mobile · Fonctionnement interne

Qu'est-ce que React Native ?
  • Framework créé par Meta (Facebook) en 2015 — développer iOS & Android avec un seul codebase JavaScript
  • Les composants sont compilés en composants natifs réels — pas une WebView comme Cordova ou Ionic
  • Si tu connais React, tu connais déjà ~70 % de React Native : mêmes hooks, même JSX, mêmes props
  • Les 30 % restants : composants différents, Flexbox mobile, APIs natives (caméra, stockage, notifications…)
💡 React Native n'est pas du React dans une WebView. L'UI est vraiment native. Performances proches d'une app Swift ou Kotlin.

🧩 Les composants React Native
💡 Analogie

Un composant, c'est comme une brique de Lego. Chaque brique a une forme (sa structure JSX) et peut recevoir des couleurs différentes (ses props). Tu assembles plusieurs briques pour construire quelque chose de plus grand — et tu peux réutiliser la même brique autant de fois que tu veux.

1

Une fonction JS qui retourne de l'interface

Un composant React Native est simplement une fonction JavaScript qui retourne du JSX — le code qui décrit ce que l'écran doit afficher.

2

On l'utilise comme une balise HTML

Une fois défini, on place le composant partout avec <MonComposant />. Chaque utilisation est une instance indépendante.

3

Il reçoit des données via les props

Les props sont les paramètres du composant. Le parent passe les données, l'enfant les affiche. C'est toujours dans ce sens — jamais l'inverse directement.

4

Il peut avoir sa propre mémoire avec useState

Si le composant doit retenir quelque chose entre les rendus (un compteur, si un menu est ouvert…), il utilise le hook useState.

Structure minimale d'un composant
import { View, Text } from 'react-native';

const MonComposant = () => {
  return (
    <View>
      <Text>Bonjour depuis MonComposant</Text>
    </View>
  );
};
export default MonComposant;

🔁 useState — Donner une mémoire au composant
💡 Analogie

Imagine un tableau blanc dans une salle de classe. useState, c'est ce tableau. La valeur actuelle = ce qui est écrit dessus. setState = effacer et réécrire. Dès que tu réécris, React détecte le changement et rafraîchit l'affichage automatiquement.

1

Déclarer — const [valeur, setValeur] = useState(initial)

valeur = la valeur actuelle. setValeur = la fonction pour la changer. useState(0) = la valeur démarre à 0.

2

Lire — {'{ valeur }'} dans le JSX

On affiche l'état dans le JSX avec des accolades. React le met à jour automatiquement à chaque changement.

3

Modifier — toujours via le setter

Ne jamais écrire valeur = 5. React ne verra pas ce changement. Toujours passer par setValeur(5) pour déclencher un re-render.

4

React re-rend le composant automatiquement

Dès que tu appelles le setter, React ré-exécute la fonction du composant et met à jour l'UI. Résultat visible immédiatement sur l'écran.

🚫
Règle absolue : ne jamais muter l'état directement.
count = count + 1 → React ne voit rien, l'UI ne change pas.
setCount(count + 1) → React détecte le changement et re-render.

Exemples — cliquer pour développer :

import { useState } from 'react';
const Compteur = () => {
  const [count, setCount] = useState(0); // démarre à 0
  return (
    <View>
      <Text>{'{'}count{'}'}</Text>
      <TouchableOpacity onPress={() => setCount(count - 1)}><Text>−</Text></TouchableOpacity>
      <TouchableOpacity onPress={() => setCount(count + 1)}><Text>+</Text></TouchableOpacity>
    </View>
  );
};
const [visible, setVisible] = useState(false);

<TouchableOpacity onPress={() => setVisible(!visible)}>
  <Text>{visible ? '▲ Masquer' : '▼ Afficher'}</Text>
</TouchableOpacity>

{visible && <Text>Contenu révélé !</Text>}
const [prenom, setPrenom] = useState('');

<TextInput
  value={prenom}
  onChangeText={(texte) => setPrenom(texte)}
  placeholder="Ton prénom..."
/>
<Text>Bonjour, {'{'}prenom || 'étranger'{'}'} !</Text>
⚠️onChangeText reçoit directement la chaîne — pas de e.target.value comme en React web !

🎨 StyleSheet.create() — Fondamentaux
💡 Analogie

StyleSheet.create(), c'est comme définir des classes CSS dans un fichier .css séparé. Au lieu de mettre le style en ligne sur chaque balise, tu définis des objets de style nommés et tu les réutilises. Les propriétés s'écrivent en camelCase et les valeurs sont des nombres en dp — pas de px, rem ou em.

❌
Style inline — à éviter
Recréé à chaque rendu
<View style={{
  backgroundColor: 'blue',
  padding: 20, borderRadius: 5
}}>
✅
StyleSheet.create()
Optimisé, validé, réutilisable
const styles = StyleSheet.create({
  card: {
    backgroundColor: 'blue',
    padding: 20, borderRadius: 5
  }
});

Exemples — cliquer pour développer :

card: {
  backgroundColor: '#1e293b', borderRadius: 12, padding: 16,
  borderLeftWidth: 4, borderLeftColor: '#3b82f6',
  // Ombres iOS
  shadowColor: '#000', shadowOffset: { width:0, height:3 },
  shadowOpacity: 0.2, shadowRadius: 5,
  // Ombre Android
  elevation: 4,
}
// Combiner styles de base + style conditionnel
<View style={[
  styles.card,               // toujours appliqué
  active && styles.cardActive  // seulement si active=true
]}>

// Couleur dynamique via prop
<View style={[styles.badge, { backgroundColor: color }]} />
import { Platform } from 'react-native';

header: {
  paddingTop: Platform.select({
    ios: 44,      // notch / Dynamic Island

📐 Flexbox Mobile — column par défaut
💡 Analogie

Flexbox, c'est une boîte qui organise ses enfants. La direction par défaut en React Native est verticale (column) — les éléments s'empilent comme des livres sur une étagère. Pour les mettre côte à côte, tu passes en row.

Exemples — cliquer pour développer :

// 1. Plein écran
screen: { flex: 1, backgroundColor: '#0f172a' }
// 2. Centrer horizontal + vertical
centered: { flex: 1, alignItems: 'center', justifyContent: 'center' }
// 3. Layout header + contenu + footer
screen:  { flex: 1 }, header: { height: 60 },
content: { flex: 1 }, footer: { height: 70 }
// 4. Éléments côte à côte avec espace entre
row: { flexDirection: 'row', justifyContent: 'space-between', alignItems: 'center' }
// 5. Grille 2 colonnes
grid: { flexDirection: 'row', flexWrap: 'wrap' },
cell: { width: '48%', margin: '1%' }

Question 1 / 10 · Architecture
Quel est le rôle principal du Bridge / JSI dans React Native ?
Le Bridge est le canal de communication asynchrone avec sérialisation JSON. JSI le remplace par une interface C++ directe. C'est lui qui permet au code JS de déclencher des mises à jour dans le thread natif.
Question 2 / 10 · Composants
Ce code cause un crash. Quelle est l'erreur ?
<View>Bienvenue sur MyExoApp</View>
Tout texte DOIT être dans <Text>. L'erreur : "Text strings must be rendered within a <Text> component". Correction : <View><Text>Bienvenue</Text></View>
Question 3 / 10 · Composants
Quel composant React Native est optimisé pour afficher de grandes listes de données ?
FlatList est virtualisée — elle ne rend que les éléments visibles. ScrollView charge tout en mémoire (mauvais pour les grandes listes). SectionList est une variante avec groupements par sections.
Question 4 / 10 · Composants
Dans le composant Player (T1-Exo04), quelle syntaxe JSX permet d'afficher du contenu différent selon une condition ?
Dans le JSX (entre les balises), on ne peut pas utiliser if/else directement. L'opérateur ternaire {'{ condition ? : }'} est le pattern standard pour le rendu conditionnel en React Native.
Question 5 / 10 · Styles
Quelle est la valeur correcte pour définir une taille de police en React Native ?
Les valeurs sont des dp (density-independent pixels). Pas de px, rem, em. Les propriétés sont en camelCase. '16px' et '1rem' sont invalides et causent une erreur.
Question 6 / 10 · Styles
Pour une ombre portée qui fonctionne sur iOS et Android, quelle combinaison utiliser ?
iOS utilise shadow* (shadowColor, shadowOffset, shadowOpacity, shadowRadius). Android utilise elevation. boxShadow n'existe pas en React Native. Les deux s'écrivent dans le même objet — chaque OS ignore les propriétés qui ne lui appartiennent pas.
Question 7 / 10 · Flexbox
Dans les exercices de zoning (T2-Exo04/05), pourquoi utilise-t-on flex: 1 sur le conteneur racine ?
flex: 1 signifie "prends tout l'espace disponible dans le parent". Sur le conteneur racine, c'est l'équivalent de height: 100vh en CSS. height: '100%' ne fonctionne pas sans un parent avec une hauteur définie.
Question 8 / 10 · Flexbox
Pour afficher deux blocs côte à côte (T2-Exo02), quelle propriété Flexbox est indispensable ?
flexDirection: 'row' est la propriété clé pour aligner les éléments sur l'axe horizontal. Par défaut en React Native, flexDirection vaut 'column' (les éléments s'empilent verticalement). display: 'flex' n'est pas nécessaire en RN — Flexbox est toujours actif.
Question 9 / 10 · useState
Pourquoi ne faut-il jamais écrire count = count + 1 pour modifier un état React ?
React suit les changements via les setters uniquement. Une mutation directe ne notifie pas React → l'UI ne se met pas à jour. Toujours utiliser setCount(count + 1).
Question 10 / 10 · Styles dynamiques
Quelle syntaxe applique un style conditionnel en plus d'un style de base ?
Le tableau [styles.base, condition && styles.extra] est le pattern standard. Le dernier style prend la priorité. L'option A bascule entre deux styles sans cumuler. Il n'existe pas de StyleSheet.merge().

Glossaire — Termes clés React Native J1

Architecture · Composants · Styles · Outils

🏗 Architecture & Moteur

Thread Architecture
Fil d'exécution indépendant. RN utilise deux threads principaux : le thread JS (logique, hooks, fetch…) et le thread natif (rendu UI iOS/Android).
setCount() s'exécute sur le thread JS → Bridge → thread natif met à jour l'affichage.
Bridge Architecture
Canal de communication asynchrone entre thread JS et thread natif dans l'ancienne architecture. Messages sérialisés en JSON.
Limitation : latence pour les interactions très fréquentes (animations).
JSI — JavaScript Interface Architecture
Nouvelle couche C++ qui permet au JS d'accéder directement aux objets natifs sans sérialisation JSON. Beaucoup plus rapide.
Résultat : animations 60/120fps, interactions synchrones.
Hermes Moteur JS
Moteur JS optimisé par Meta pour RN. Compile le code JS en bytecode à l'avance (AOT), réduit le démarrage et la mémoire.
Activé par défaut depuis RN 0.70.
React Native Runtime Architecture
L'environnement d'exécution complet : moteur JS (Hermes), bindings natifs, système de modules, APIs exposées.
Inclut : timers, modules natifs, système d'événements, gestionnaire de layout.
Metro Bundler Outil
Bundler JS de RN (≈ Webpack). Transforme JSX/TS en JS pour le device. Lancé par npx expo start.
Serveur port 8081. Envoie le bundle à Expo Go via QR code.

⚛️ JSX, Composants & React

JSX Syntaxe
Extension de syntaxe JS qui permet d'écrire du HTML-like. Compilé en React.createElement(). Composants différents entre React et React Native.
<View> → React.createElement(View, {})
Composant UI Composants
Fonction retournant du JSX, compilée en widget natif iOS/Android. Peut recevoir des props et gérer un état local.
Car, House, Person, Player sont des composants UI de MyExoApp.
Props Composants
Propriétés passées parent → enfant. Immuables côté enfant. Pour remonter une info, passer une fonction callback en prop.
title="Peugeot" est une prop.
État (State) React
Donnée locale à un composant pouvant changer. Chaque modification via le setter déclenche un re-render.
const [score, setScore] = useState(100)
Hook React
Fonction spéciale React commençant par use. Doit toujours être appelée en haut du composant, jamais dans une condition.
useState, useEffect, useCallback, useMemo, useContext.
Re-render React
Réexécution de la fonction du composant suite à un changement d'état ou de props. React met à jour uniquement ce qui a changé.
setScore() → re-render → nouveau score affiché.

🎨 Styles & Flexbox

StyleSheet Style
StyleSheet.create() valide les propriétés, optimise et enregistre les styles une seule fois. Équivalent des classes CSS en objets JS.
Valeurs en dp, pas en px.
dp — Density-independent pixel Style
Unité abstraite qui s'adapte à la densité de l'écran. 1dp = 2 ou 3 pixels physiques sur écran haute densité.
fontSize: 16 → même taille visuelle sur iPhone SE et Samsung S24.
elevation Style Android
Propriété Android (0-24) qui crée une ombre portée. N'a aucun effet sur iOS — à combiner avec shadow*.
elevation: 4 ≈ ombre légère sur Android.
flexDirection Flexbox
Définit l'axe principal de disposition. column par défaut en RN (contrairement à row en CSS web). Changer en 'row' pour aligner côte à côte.
T2-Exo02 : flexDirection: 'row' pour les boutons côte à côte.
SafeAreaView Composant
Adapte le contenu pour éviter les zones système : notch iOS, Dynamic Island, status bar Android. À utiliser comme conteneur racine de chaque écran.
Utilisé dans les exercices de zoning T2-Exo04/05.
Platform.select() Style
Retourne une valeur différente selon la plateforme. Utilisé dans les StyleSheets pour gérer les différences iOS/Android.
Platform.select({'{ ios: 44, android: 24 }'})

🛠 Outils & Déploiement

Expo Outil
Plateforme simplifiant le développement RN. SDK complet, CLI, app Expo Go, EAS Build. Couvre 95 % des besoins.
npx create-expo-app MyExoApp — tout est prêt en 2 min.
Expo Go Outil
App mobile qui teste une app Expo via QR code. Charge le bundle JS en temps réel depuis le serveur de dev.
Voir immédiatement chaque exercice de MyExoApp sur son téléphone.
EAS Build Déploiement
Service cloud Expo qui compile l'app en APK/IPA sur des serveurs distants. Build sans Xcode/Android Studio local.
Abordé en J4 pour générer l'APK du projet fil rouge.

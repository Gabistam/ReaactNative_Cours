S3 · React Navigation

Installation · Stack Navigator · Tab Navigator · Navigation imbriquée · Paramètres

📦 Installation & configuration
💡 Analogie

React Navigation, c'est le routeur de ton app mobile — comme React Router pour le web. Il gère l'historique des écrans, les transitions, le bouton retour Android, et le swipe iOS. Sans lui, ton app n'est qu'un seul écran sans lien entre eux.

Installer le core : npx expo install @react-navigation/native
Installer les dépendances Expo : npx expo install react-native-screens react-native-safe-area-context
Installer Stack : npm install @react-navigation/native-stack
Installer Tab : npm install @react-navigation/bottom-tabs
Installer Ionicons (icônes) : npm install @expo/vector-icons — déjà inclus dans Expo !
Envelopper App.js dans <NavigationContainer>
App.js — structure de base avec NavigationContainer
import { NavigationContainer } from '@react-navigation/native';
import { AppNavigator } from './navigation/AppNavigator';

export default function App() {
  return (
    <NavigationContainer>  // ← wrapper obligatoire, une seule fois
      <AppNavigator />
    </NavigationContainer>
  );
}
⚠️NavigationContainer doit englober toute l'app — une seule fois, dans App.js. Ne jamais le mettre dans un composant enfant.
📚 Stack Navigator — navigation en pile
💡 Analogie

Le Stack Navigator fonctionne comme une pile de cartes. Chaque navigate() empile un écran dessus. goBack() enlève la carte du dessus et revient à la précédente. Sur iOS, c'est le swipe de droite. Sur Android, c'est le bouton retour natif.

navigation/AppNavigator.js — Stack de base
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { HomeScreen, DetailScreen } from '../screens';

const Stack = createNativeStackNavigator();

export function AppNavigator() {
  return (
    <Stack.Navigator
      initialRouteName="Home"
      screenOptions={{ headerStyle: { backgroundColor: '#0f172a' },
                       headerTintColor: '#f1f5f9', headerTitleStyle: { fontWeight: '700' } }}
    >
      <Stack.Screen name="Home"   component={HomeScreen}   options={{ title: 'Accueil' }}  />
      <Stack.Screen name="Detail" component={DetailScreen} options={{ title: 'Détail' }}  />
    </Stack.Navigator>
  );
}

Exemples — cliquer pour développer :

// Dans HomeScreen — envoyer des données vers DetailScreen
const HomeScreen = ({ navigation }) => (
  <TouchableOpacity onPress={() =>
    navigation.navigate('Detail', {
      themeId: 'T1',
      title: 'Composants & JSX',
      count: 6
    })
  }>
    <Text>Voir le thème</Text>
  </TouchableOpacity>
);

// Dans DetailScreen — recevoir les données
const DetailScreen = ({ route, navigation }) => {
  const { themeId, title, count } = route.params;
  return (
    <View>
      <Text>{'{title}'} — {'{count}'} exercices</Text>
      <TouchableOpacity onPress={() => navigation.goBack()}>
        <Text>← Retour</Text>
      </TouchableOpacity>
    </View>
  );
};
// Options statiques dans le Navigator
<Stack.Screen
  name="Detail"
  component={DetailScreen}
  options={{ title: 'Détail', headerBackTitle: 'Retour' }}
/>

// Options dynamiques depuis le composant (titre basé sur les params)
<Stack.Screen
  name="Detail"
  component={DetailScreen}
  options={({ route }) => ({ title: route.params.title })}
/>

// Modifier le header depuis le composant lui-même
useEffect(() => {
  navigation.setOptions({ title: data.name });
}, [data]);

🗂 Bottom Tab Navigator — navigation par onglets
💡 Analogie

Le Tab Navigator, c'est la barre de navigation du bas que tu vois dans Instagram, Twitter ou YouTube. Chaque onglet est un écran indépendant. Contrairement au Stack, les onglets coexistent en mémoire — passer de l'un à l'autre est instantané.

Tab Navigator avec Ionicons
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { Ionicons } from '@expo/vector-icons';

const Tab = createBottomTabNavigator();

export function TabNavigator() {
  return (
    <Tab.Navigator
      screenOptions={({ route }) => ({
        tabBarIcon: ({ focused, color, size }) => {
          const icons = {
            Accueil:  focused ? 'home'          : 'home-outline',
            Explorer: focused ? 'compass'       : 'compass-outline',
            Profil:   focused ? 'person'        : 'person-outline',
          };
          return <Ionicons name={icons[route.name]} size={size} color={color} />;
        },
        tabBarActiveTintColor:   '#34d399',
        tabBarInactiveTintColor: '#64748b',
        tabBarStyle: { backgroundColor: '#1e293b', borderTopColor: '#334155' },
        headerStyle: { backgroundColor: '#0f172a' },
        headerTintColor: '#f1f5f9',
      })}
    >
      <Tab.Screen name="Accueil"  component={HomeScreen}    />
      <Tab.Screen name="Explorer" component={ExploreScreen} />
      <Tab.Screen name="Profil"   component={ProfileScreen}  />
    </Tab.Navigator>
  );
}

S4 · useEffect & Fetch API

useEffect · fetch · async/await · états loading/error/data · FlatList dynamique

⚡ useEffect — Déclencher du code après le rendu
💡 Analogie

useEffect, c'est comme un gestionnaire d'événements de cycle de vie. Il te permet de déclencher du code après que le composant s'est affiché — par exemple, charger des données depuis une API. Sans useEffect, ton fetch s'exécuterait à chaque re-render, en boucle infinie.

1

Le composant se rend une première fois — souvent vide

React affiche d'abord le composant avec les données initiales (souvent [] ou null). L'utilisateur voit un état de chargement.

2

useEffect s'exécute après ce premier rendu

Le code dans useEffect tourne après l'affichage. C'est là qu'on lance le fetch(). La dépendance [] signifie "exécuter une seule fois, au montage".

3

Les données arrivent → setState() → re-render

Quand le fetch réussit, on met à jour l'état via setData(). React re-render automatiquement avec les nouvelles données.

4

Le tableau de dépendances contrôle quand ré-exécuter

[] = une seule fois au montage. [id] = à chaque changement de id. Vide (sans tableau) = à chaque re-render (⚠ boucle infinie possible).

useEffect — syntaxe complète
import { useState, useEffect } from 'react';

const MonComposant = ({ id }) => {
  const [data,    setData]    = useState(null);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    fetchData();   // lancer le fetch
  }, [id]);        // re-exécuter si id change

  const fetchData = async () => {
    try {
      setLoading(true);
      const res  = await fetch(`https://api.example.com/items/${'{id}'}`);
      const json = await res.json();
      setData(json);
    } catch (err) {
      setError('Erreur de chargement');
    } finally {
      setLoading(false);
    }
  };

  if (loading) return <ActivityIndicator />;
  if (error)   return <Text>{'{error}'}</Text>;
  return       <Text>{'{data.name}'}</Text>;
};

App.js — commutation AuthStack / AppStack (mock J2)
import { useState } from 'react';

import { AuthStack } from './navigation/AuthStack';
import { AppStack } from './navigation/AppStack';

export default function App() {
  // J2 : mock — J3 : remplacé par onAuthStateChanged()
  const [isLoggedIn, setIsLoggedIn] = useState(false);

  return (
    <NavigationContainer>
      {isLoggedIn
        ? <AppStack  onLogout={() => setIsLoggedIn(false)} />
        : <AuthStack onLogin={() => setIsLoggedIn(true)}  />
      }
    </NavigationContainer>
  );
}

10 QCM — Vérification des acquis J2

Navigation · useEffect · Fetch · Architecture

📝Sélectionne une réponse par question puis clique sur "Valider mes réponses".
Question 1 / 10 · Navigation
Quel composant doit impérativement envelopper toute l'application pour que React Navigation fonctionne ?
NavigationContainer est le wrapper obligatoire qui fournit le contexte de navigation à toute l'app. Il ne doit apparaître qu'une seule fois, dans App.js. Sans lui, aucun navigator ne fonctionne.
Question 2 / 10 · Stack Navigator
Comment passer des paramètres d'un écran A vers un écran B en Stack Navigation ?
Le deuxième argument de navigate() est un objet de paramètres. L'écran destination les reçoit via route.params. C'est le mécanisme standard — pas de contexte ni de stockage externe nécessaire pour un passage simple.
Question 3 / 10 · Stack Navigator
Quelle différence fondamentale entre navigate('Ecran') et replace('Ecran') ?
navigate() empile l'écran dans la pile — le bouton retour est disponible. replace() remplace l'écran courant sans l'empiler — utile après une authentification réussie pour que l'utilisateur ne puisse pas "revenir" à l'écran de login.
Question 4 / 10 · Tab Navigator
Quelle est la différence principale entre Stack Navigator et Tab Navigator en termes d'état des écrans ?
Dans un Tab Navigator, tous les onglets restent montés en mémoire une fois visités — passer de l'un à l'autre est instantané car le rendu est déjà fait. Dans un Stack, les écrans dépilés (goBack()) sont détruits. C'est pour ça que les onglets sont plus réactifs.
Question 5 / 10 · useEffect
Que signifie le tableau [] (vide) comme deuxième argument de useEffect ?
[] signifie "aucune dépendance" — l'effet ne dépend de rien qui change. Il tourne donc une seule fois, juste après le premier rendu (montage). C'est le pattern standard pour un fetch initial. Sans tableau du tout, l'effet tourne après chaque rendu → risque de boucle infinie.
Question 6 / 10 · useEffect
Pourquoi ne pas écrire directement useEffect(async () => {'{ await fetch(...) }'}, []) ?
useEffect attend que sa callback retourne soit rien, soit une fonction de cleanup. Une fonction async retourne toujours une Promise — ce qui crée des comportements inattendus. Pattern correct : définir une fonction fetchData async à l'intérieur et l'appeler immédiatement.
Question 7 / 10 · Fetch
Dans quel ordre s'enchaînent correctement les 3 états lors d'un fetch réussi ?
L'ordre est : setLoading(true) → afficher spinner → fetch() → setData(json) → setLoading(false) → afficher les données. Le bloc finally garantit que loading repasse à false même en cas d'erreur.
Question 8 / 10 · Fil rouge
Dans App Quiz (J2), la commutation AuthStack/AppStack est gérée par un booléen mock. En J3, que le remplacera ?
onAuthStateChanged() est le listener Firebase qui écoute les changements d'état d'authentification en temps réel. Il retourne l'utilisateur courant (ou null si déconnecté) — on remplace isLoggedIn par !!user dans le rendu conditionnel.
Question 9 / 10 · Navigation imbriquée
Dans le pattern Stack dans Tab, si l'utilisateur navigue de ThemeList → QuizScreen (dans l'onglet Home), que se passe-t-il s'il tape sur l'onglet Profil puis revient sur l'onglet Home ?
Le Tab Navigator conserve l'état de chaque onglet en mémoire. La Stack de l'onglet Home (avec QuizScreen en haut de pile) est préservée quand l'utilisateur bascule vers Profil. Il retrouve exactement là où il en était à son retour.
Question 10 / 10 · Architecture
Quelle est la bonne pratique pour les fichiers de navigation dans App Quiz ?
Séparer la navigation dans /navigation est une convention standard. AppNavigator orchestre la commutation, AuthStack gère les écrans d'auth, AppStack gère les écrans de l'app connectée. Ça facilite l'évolution (ajout de Firebase J3) sans toucher aux écrans.

Glossaire — Termes clés J2

Navigation · useEffect · Fetch · Architecture

🧭 React Navigation

NavigationContainer Navigation
Composant wrapper obligatoire qui fournit le contexte de navigation à toute l'app. Se place une seule fois dans App.js. Sans lui, aucun navigator ne fonctionne.
Équivalent du <BrowserRouter> de React Router web.
Stack Navigator Navigation
Gère une pile d'écrans. navigate() empile un écran, goBack() le dépile. Fournit un header automatique avec bouton retour sur iOS/Android.
Utilisé pour les flux linéaires : liste → détail → sous-détail.
Bottom Tab Navigator Navigation
Barre de navigation en bas de l'écran avec plusieurs onglets. Chaque onglet conserve son état en mémoire une fois monté.
Pattern omniprésent dans les apps mobiles (Instagram, YouTube…).
route.params Navigation
Objet contenant les paramètres passés lors d'un navigate('Ecran', params). Accessible dans le composant destination via la prop route.
const {'{'} id, title {'}'} = route.params;
navigation.navigate() Navigation
Navigue vers un écran en l'empilant dans la Stack. Si l'écran est déjà dans la pile, navigue vers l'instance existante. Peut passer des paramètres en second argument.
Vs replace() qui remplace sans empiler.
navigation imbriquée Navigation
Pattern qui combine plusieurs navigators : un Tab contenant des Stacks dans chaque onglet. Permet une navigation riche et indépendante par onglet.
Pattern utilisé dans App Quiz : Tab (Home Stack + Profil).

⚡ Hooks & Asynchrone

useEffect Hook React
Hook qui exécute du code après le rendu. Tableau de dépendances contrôle quand ré-exécuter : [] = une fois au montage, [id] = à chaque changement de id.
Sert principalement aux appels API, abonnements, timers.
async/await JavaScript
Syntaxe pour gérer les opérations asynchrones de façon lisible. await attend la résolution d'une Promise sans bloquer le thread principal.
const data = await fetch(url).then(r => r.json())
try/catch/finally JavaScript
Gestion des erreurs dans un code async. try = tentative, catch = en cas d'erreur, finally = toujours exécuté (ex: setLoading(false)).
finally garantit que le spinner disparaît même si le fetch échoue.
ActivityIndicator Composant RN
Composant natif React Native qui affiche un spinner de chargement. Afficher pendant loading=true, masquer quand les données arrivent.
<ActivityIndicator size="large" color="#34d399" />
RefreshControl Composant RN
Composant qui ajoute le geste pull-to-refresh sur une FlatList ou ScrollView. Déclenche la callback onRefresh quand l'utilisateur tire vers le bas.
Combiné avec un état refreshing booléen pour le spinner.
fetch() API Web
API standard JavaScript pour les requêtes HTTP. Retourne une Promise. Disponible nativement en React Native via Hermes — pas besoin d'axios par défaut.
const res = await fetch(url); const json = await res.json();

🏗 Architecture

AuthStack Architecture
Stack Navigator dédié aux écrans d'authentification (Login, Register). Affiché quand l'utilisateur n'est pas connecté. Remplacé par AppStack à la connexion.
Pattern universel dans toutes les apps avec auth.
AppStack Architecture
Navigator principal affiché quand l'utilisateur est connecté. Contient généralement un Tab Navigator avec les écrans de l'app.
Commutation AuthStack ↔ AppStack via onAuthStateChanged en J3.
Mock / Données mockées Architecture
Données ou comportements simulés en attendant l'implémentation réelle. En J2, isLoggedIn est un mock qui sera remplacé par Firebase Auth en J3.
Permet de construire et tester la navigation sans backend.
